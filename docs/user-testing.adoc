== 1. Introduction

This document outlines the User Acceptance Testing (UAT) plan for the AsciiDoc XML Converter. The goal is to verify that the system meets user requirements, functions correctly across defined scenarios, and provides a seamless user experience.

=== 1.1 Objectives
* Verify the core functionality: AsciiDoc to XML/HTML conversion.
* Validate the web interface usability and responsiveness.
* Ensure the Batch Processing feature works as expected.
* Confirm the robustness of the File Browser and Example loading.
* Test edge cases and error handling.

=== 1.2 Scope
* **In Scope**: Web interface (`/`, `/batch`, `/browse`), API endpoints, CLI usage (if applicable for UAT), standard AsciiDoc syntax, and custom extensions defined in the project.
* **Out of Scope**: Performance stress testing (load testing), unit testing (covered in development), and internal code structure.

== 2. Test Environment

* **Browser**: Latest versions of Chrome, Firefox, and Edge.
* **OS**: Windows, Linux, macOS (as applicable).
* **Server**: Local instance running on `http://localhost:8005`.
* **Test Data**:
** `examples/comprehensive.adoc` (Complex syntax)
** `examples/rfc791/` (Multi-file, linked content)
** Custom invalid files for error testing.

== 4. Acceptance Criteria

* All **Critical** (TC-001 to TC-004, TC-010, TC-011) tests must pass.
* No blocking bugs that prevent the user from completing a core workflow (Conversion, Browsing).
* UI must be responsive and usable on a standard 1080p screen.
